# 離乳食プランナー アプリ設計メモ

作成日：2026-03-30
想定スタート月齢：中期（7〜8ヶ月）を中心に、初期〜完了期に対応

---

## コンセプト

**「今日、何を食べさせればいいか迷わない」**

毎日の献立を考える認知的コストをゼロに近づける。
食材の進捗管理・AIによる献立生成・アレルギー記録を一元化する。

---

## ターゲットユーザー

- 離乳食を始めて1〜3ヶ月の親（初期後半〜中期）
- スマホメインで隙間時間に記録したい
- 「育児書を読む時間はないが、安全に進めたい」
- 特に中期特有の悩み：食べ物の固さ、2回食のタイミング、食材のバリエーション不足

---

## 解決する課題（中期の親が特に直面するもの）

1. **「これ食べさせていいっけ？」問題** — 月齢ごとにOKな食材が違い、都度調べるのが面倒
2. **献立のマンネリ** — 安全な食材しか使わず、毎日同じものを出しがち
3. **食べた記録がバラバラ** — メモ帳・写真・頭の中に分散していてアレルギー確認が難しい
4. **栄養バランスの不安** — 炭水化物に偏っていないか、たんぱく質が足りているか把握できない

---

## 機能設計

### Phase 1（MVP）— まず作るもの

#### 1. 赤ちゃんプロフィール
- 生年月日を入力 → 月齢を自動計算
- 離乳食ステージを自動判定（初期/中期/後期/完了期）
- アレルギー食材の登録（卵・小麦・乳製品・ナッツ類など）

#### 2. 食材チェックリスト
- 月齢に対応した食材をカテゴリ別に一覧表示
  - カテゴリ：炭水化物 / タンパク質 / 野菜・果物 / 乳製品
- 各食材に3段階ステータス
  - ✅ 試した（反応OK）
  - ⚠️ 試したが食べなかった
  - ❌ アレルギー反応あり
  - 未マーク = まだ試していない
- 「次に試したい食材」としてブックマーク機能
- 進捗バー：「中期対応食材のうち〇〇%試した」

#### 3. 今日の食事記録
- 朝・昼・夜の3ブロック（将来的に補食追加）
- 食材をタップして追加するUIで入力を素早く
- 自由テキストでメモ（「よく食べた」「吐き出した」など）
- 写真添付オプション

#### 4. AI献立生成
- 「今週の献立を作って」ボタン1タップで5〜7日分を生成
- 生成ロジック（AIへの指示内容）：
  - 現在の月齢・ステージ
  - アレルギー食材を除外
  - まだ試していない食材を週1〜2個自然に組み込む
  - 直近3日間と食材が被りすぎないよう調整
  - 固さの説明文を付与（「豆腐程度の固さに」など）
- 生成結果をカレンダービューで表示
- 個別の食事を入れ替え・削除可能

---

### Phase 2（次のフェーズ）

#### 5. 買い物リスト自動生成
- 週間献立から必要な食材を抽出
- 「家にある」チェックで不要なものを除外
- 食材をカテゴリ別にグルーピング

#### 6. 栄養バランスビジュアライザー
- 今週食べた食材を炭水化物・タンパク質・野菜で集計
- 簡易グラフ表示
- 「今週タンパク質が少なめです。豆腐や白身魚を取り入れてみましょう」などAIコメント

#### 7. 月次サマリー
- 「〇〇ちゃん、今月初めて食べたもの」リスト
- 食べた食材数の推移グラフ
- PDF/画像でダウンロード → SNSシェア・母子手帳補完に使える

```
ホーム
├── 今日の献立（朝/昼/夜）
├── 今日食べたものを記録
└── 今週の食材バランスサマリー

食材ライブラリ
├── カテゴリタブ（炭水化物/タンパク質/野菜/乳製品）
├── 各食材のステータス管理
└── 未試食材の一覧

献立プランナー
├── 週間カレンダービュー
├── AI献立生成ボタン
└── 手動編集

記録カレンダー
└── 日付別の食事記録閲覧

設定
├── 赤ちゃん情報（生年月日）
├── アレルギー設定
└── ステージ手動調整
```

## データ設計

### Baby（赤ちゃんプロフィール）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | ユニークID |
| name | string | 赤ちゃんの名前 |
| birthDate | date | 生年月日 |
| allergies | string[] | アレルギー食材ID配列 |
| stage | enum | 自動計算 or 手動設定 |

### FoodItem（食材マスタ）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 食材ID |
| name | string | 食材名（日本語） |
| category | enum | 炭水化物/タンパク質/野菜/果物/乳製品 |
| availableFrom | enum | 初期/中期/後期/完了期 |
| allergyRisk | enum | 高/中/低 |
| prepNote | string | 中期での調理方法の説明 |

### FoodLog（食事記録）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | 記録ID |
| babyId | string | 赤ちゃんID |
| date | date | 記録日 |
| mealType | enum | 朝/昼/夜/補食 |
| foods | FoodItem[] | 食べたもの |
| reaction | enum | OK/食べなかった/アレルギー疑い |
| note | string | メモ |

### MealPlan（AI生成献立）
| フィールド | 型 | 説明 |
|---|---|---|
| id | string | プランID |
| babyId | string | 赤ちゃんID |
| weekStartDate | date | 週の開始日 |
| days | DayPlan[] | 日ごとの献立 |
| generatedAt | timestamp | 生成日時 |

## AI活用ポイント

### 週次献立生成（メイン機能）
プロンプト設計のイメージ：

```
現在の月齢：7ヶ月（離乳食中期・もぐもぐ期）
アレルギー除外：卵
直近3日間に使った食材：お粥、にんじん、豆腐、ほうれん草
まだ試していない中期対応食材：しらす、ブロッコリー、さつまいも、バナナ

上記を踏まえ、5日分の2回食献立を提案してください。
・各食事に主食・野菜・タンパク質を含めてください
・新食材は1日1種類まで、少量から始めること
・固さは舌でつぶせる固さ（豆腐程度）を基準に
```

### 食材調理アドバイス
「ブロッコリーを中期の赤ちゃん向けに」→ 下処理・調理法・固さの目安を返す

### 食べない時の対処アドバイス
食べなかった食材と状況を入力 → 原因候補と対処法を提案

## 競合分析

| アプリ | 強み | 弱み（差別化余地） |
|---|---|---|
| ぴよログ | 成長記録で圧倒的シェア | 食事管理・献立機能は薄い |
| 和光堂アプリ | 公式レシピ豊富 | 自社製品誘導、AI機能なし |
| パルシステム | 食材連携 | 会員限定、汎用性なし |
| 汎用カレンダー/メモ | 自由度高い | 離乳食専用の設計がない |

**空白地帯：「食材進捗管理 × AI献立生成」の組み合わせ**
